# Changelog

App versions follow `v0.0.x` — one bump per shipped milestone (not per commit).
When shipping a main feature: add an entry here, bump `version` in
`package.json`, and mirror a short 繁中 summary in `web/src/version.ts`
(`APP_VERSION` + `VERSION_HISTORY`), which drives the in-app 關於 page.

## v0.0.10 — 2026-07-12

- AI food parsing is now **grounded** in the 衛生福利部食品藥物管理署「食品營養
  成分資料庫」(TFND) open dataset (data.gov.tw #8543, Open Government Data
  License v1.0). Instead of the model inventing per-100g numbers, it now
  estimates only the portion (grams) and a canonical food term; the server
  looks the term up in a bundled 1,793-food Taiwan table and recomputes protein
  /kcal from the official per-100g value × grams. Unmatched foods fall back to
  the model's own estimate.
- Each parsed item is tagged `source` (`db` grounded / `ai` estimated) and, when
  grounded, the matched official food name — surfaced as a 資料庫 / AI 估 badge in
  the confirm/edit view so the user can see and correct any mismatch.
- Reference data lives in `src/data/tw-food.ts` (a compact tab-encoded string,
  ~74KB, gzips to ~30KB), regenerated by `scripts/build-food-db.mjs` from the
  FDA dump. The raw 226k-row long-format dataset is pivoted to per-food protein
  + calories, then near-duplicate sampling/cooking variants are collapsed by
  base name using the median. In-memory longest-common-substring matching lives
  in `src/data/food-db.ts` (tests in `food-db.test.ts`).

## v0.0.9 — 2026-07-11

- Workout page now leads with a "上次訓練" line showing how long ago you last
  trained (今天 / 昨天 / N 天前 + the date), backed by a new
  `GET /api/workouts/latest` endpoint (MAX training date across all exercises)
- Rest reminder: after 3 idle days the coach nudges you to train again, with
  escalating copy at 3 / 5 / 10 days. Static messages (no AI call) written in
  the user's selected coach tone 友善 / 嚴格 / 專業, styled like coach feedback

## v0.0.8 — 2026-07-11

- Public landing page at `/welcome` (no login) with a waiting-list signup —
  strangers leave their email; served self-contained by the Worker so it needs
  no Access-protected assets
- Admin 候補名單 in `#/admin`: review signups, send an invite with one tap,
  or remove an entry
- Invite emails via personal Gmail SMTP (smtp.gmail.com:465 over Cloudflare
  TCP sockets, App Password auth) — delivers to any recipient, degrades to a
  copyable invite link when `GMAIL_APP_PASSWORD` is unset or a send fails
- **Requires** a Cloudflare Access **Bypass** policy for `/welcome` and
  `POST /api/waitlist` so logged-out visitors can reach them (see README)
